# Кликер набора баллов

Интерактивное приложение-кликер, где пользователь набирает баллы кликами по изображению. При достижении пороговых значений меняется изображение, оценка и цветовая схема.

## Цель работы

Изучить жизненный цикл Activity в Android и научиться правильно обрабатывать переходы между состояниями. В результате лабораторной работы будет создано интерактивное приложение "Grade Clicker" (Кликер оценок), которое демонстрирует работу lifecycle callbacks и сохранение состояния при изменении конфигурации устройства.

## Ответы на контрольны вопросы

1. Какие три метода вызываются при запуске приложения?:
   -  `onCreate()`, `onStart()`, `onResume()`

2. Чем отличается onPause() от onStop()?:
   - `onPause()` - Activity теряет фокус, но может быть частично видна.
   - `onStop()` - Activity становиться полностью невидимой для пользователя.

3. Когда вызывается onRestart()?:
   - Вызывается после `onStop()`, когда Activity перезапускается перед тем, как снова стать видимой.

4. Что происходит при повороте экрана?:
   - Activity полностью стирается и создается заново, что приводит к потере данных без специального сохранения.

5. В чём разница между remember и rememberSaveable?:
   - `remember` - сохраняет данные только при рекомпозиции рекомпозиции.
   - `rememberSaveable` - дополнительно сохраняет данные при изменении конфигурации.

6. Почему onCreate() должен вызывать super.onCreate()?:
   - Для корректного завершения создания Activity платформой Android, иначе возникнет ошибка.

7. Для чего используется класс Log?
   - Для вывода отладочных сообщений в Logcat.

8. Какой метод вызывается только один раз за время жизни Activity?:
   -  `onCreate()` - вызывается единожды при создании Activity.
